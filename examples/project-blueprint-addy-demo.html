Addy — a tiny transformer that learns to add

A from-scratch, character-level, decoder-only transformer trained on nothing but synthetically generated addition problems. Given a prompt like 85+47=, it generates the digits of the sum one token at a time.

char-level decoder-only < 100k params ≥ 99% exact-match
01

Problem & motivation

Large language models feel like black boxes, and GPT-scale models are far too big to learn from by reading the weights. A model small enough to train on a laptop in minutes — on a task with exactly one correct answer — makes every mechanism inspectable: tokenization, embeddings, attention, autoregressive decoding, sampling, and generalization. Addition is the "hello world" of sequence reasoning: trivial to generate infinite labelled data, correctness is crisp, yet the model must still learn carry propagation — a genuine multi-step dependency, not just pattern lookup.

02

Goals & non-goals

Goals
  • Train a sub-100k-parameter char-level transformer from scratch to add two non-negative integers.
  • Reach ≥99% exact-match accuracy on held-out 2-digit sums.
  • Fully reproducible and fast — trains on CPU in minutes from a fixed seed.
  • Serve as a teaching artifact: every module is small and readable.
Non-goals
  • Not a general-purpose LLM — no natural language, no chat.
  • No subtraction, multiplication, negatives, or floats in v1.
  • Not chasing size or speed records.
  • No pretrained weights and no external datasets — data is generated on the fly.
03

Target users / personas

  • ML learner — wants to understand transformers by building the smallest real one end to end.
  • Educator — needs a runnable, inspectable demo of attention, autoregression, and sampling.
  • Tinkerer — wants to poke hyperparameters and the digit count and watch accuracy move.
04

User stories

  • As a learner, I want to type two numbers and watch the model emit the answer digit by digit, so I can see autoregressive generation in action.
  • As a learner, I want to change the sampling temperature, so I can feel the difference between greedy and sampled decoding.
  • As an educator, I want a training curve and an accuracy readout, so I can show convergence live.
  • As a tinkerer, I want to set the number of digits and dataset split, so I can test memorization vs generalization.

06

Non-functional requirements

  • Performance — trains to target in < 5 min on a modern CPU; inference < 50 ms per sum.
  • Footprint — < 100k parameters; one file, minimal deps (e.g. PyTorch or NumPy).
  • Reproducibility — a fixed seed yields identical weights and metrics.
  • Readability — each module under ~100 lines; no clever trick without a comment.
  • Accessibility — the teaching UI is keyboard-navigable.

08

Key user flows

Inference — the core "make a thing" flow

User enters a and b → encode "a+b=" → loop: forward pass, take logits at the last position, apply temperature softmax, pick the next token, append it → continue until the answer digits are emitted (least-significant first) → un-reverse and decode → show the sum.

Training

Sample a batch of sums → forward → masked cross-entropy on the answer span → backprop → optimizer step → every N steps run exact-match eval → stop when accuracy target or max steps is reached.

10

Success metrics

  • ≥99% exact-match on held-out 2-digit sums.
  • Training wall-clock < 5 min on CPU.
  • Parameter count < 100k.
  • A new user can explain autoregression after five minutes with the demo (qualitative).
11

Risks & open questions

ASSUMENon-negative integers with a fixed maximum digit count (2 in v1); inputs are zero-padded to a fixed width.
ASSUMEAnswer digits are reversed (least-significant first) during training — a documented trick to help the model learn carries; see the Advanced visual for why.
RISKThe model memorizes instead of generalizing → mitigate with a value-disjoint train/test split and an out-of-distribution digit-length eval.
RISKA model this tiny may underfit carries on longer numbers → may need an extra layer or head.
OPENWhether to add an explicit end-of-sequence token, or stop purely on answer length.

Advanced

The reverse-digit trick

Carries flow right-to-left, but the model emits the answer left-to-right. Step through a column-by-column addition in each order. In natural order the first digit it must emit depends on carries it hasn't computed yet (shown in red); in reversed order every step already has the carry it needs.
